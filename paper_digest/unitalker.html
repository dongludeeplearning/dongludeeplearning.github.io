High-Level Insights: Why it Works?

UniTalker works by abstracting the common "audio-to-motion" logic into a shared encoder while delegating format-specific details to lightweight heads. The PCA strategy normalizes the difficulty of different output formats, ensuring the shared encoder isn't biased towards easier tasks. The Pivot Identity Embedding acts like a "universal speaker" anchor, helping the model learn generalizable speech-motion correlations rather than memorizing specific dataset quirks.

# Unified Forward Pass
def forward(audio, speaker_id, dataset_id):
    # 1. Shared Audio Encoding
    feat = audio_encoder(audio)
    
    # 2. Pivot Identity Embedding (Training)
    if training and random() < pivot_prob:
        id_emb = pivot_embedding
    else:
        id_emb = speaker_embedding(speaker_id)
        
    # 3. Multi-Head Decoding
    if dataset_id == 'BIWI':
        # Output PCA components -> Vertices
        out = decoder_head_biwi(feat, id_emb)
        vertices = pca_inverse(out)
    elif dataset_id == 'ARKit':
        # Output Blendshape Weights
        out = decoder_head_arkit(feat, id_emb)
        vertices = blendshape_layer(out)
        
    return vertices

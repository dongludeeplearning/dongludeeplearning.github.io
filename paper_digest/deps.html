Methodï¼ˆæ–¹æ³•ï¼‰

DEPS addresses open-world planning challenges through interactive LLM-based planning:

  1. Descriptor Module: Summarizes execution state and failure contexts when sub-goals cannot be completed, providing rich feedback to guide plan correction.
  2. Explainer Module: Uses LLM reasoning to diagnose why plans fail by analyzing execution feedback and identifying missing prerequisites or incorrect assumptions.
  3. Re-Planner Module: Generates corrected plans by incorporating descriptor feedback and explainer insights, enabling iterative plan refinement through multiple rounds.
  4. Goal Selector: Learns to predict completion horizons for parallel sub-goals and selects the most efficient execution order based on current agent state and environmental constraints.
  5. Integration Pipeline: Combines all modules in a hierarchical architecture where initial plans are iteratively refined through execution feedback and efficiency optimization.

Why it Works?

The success of DEPS stems from principled integration of interactive planning with LLM capabilities:

  • Interactive Error Correction: The descriptor-explainer-replanner loop provides a systematic way to diagnose and fix plan failures, leveraging LLM's reasoning capabilities to understand complex execution contexts.
  • State-Aware Goal Selection: The horizon-predictive selector learns practical experience about task completion difficulty, enabling efficient parallel goal ordering that adapts to current environmental constraints.
  • Multi-Round Refinement: Allowing multiple rounds of plan correction enables progressive improvement, where each iteration builds on the insights gained from previous execution attempts.
  • Hierarchical Architecture: Separating high-level planning (LLM-based) from low-level execution (goal-conditioned controllers) allows each component to specialize while maintaining overall system coherence.
  • Zero-Shot Generalization: The framework's reliance on LLM's general knowledge and interactive learning enables strong performance without task-specific fine-tuning or demonstrations.

// Descriptor Module - Provides execution feedback
class ExecutionDescriptor:
    """Summarizes current execution state and failure contexts"""
    
    def describe_failure(self, current_plan, failed_step, inventory, environment_state):
        """Generate textual description of execution failure"""
        description = f"I failed on step {failed_step}: {current_plan[failed_step]}. "
        description += f"My inventory now has {inventory}. "
        description += f"Environment state: {environment_state}"
        return description

// Explainer Module - Diagnoses plan failures
class FailureExplainer:
    """Uses LLM to diagnose why plans fail"""
    
    def explain_failure(self, failure_description, task_goal):
        """Analyze failure and identify root causes"""
        prompt = f"Task: {task_goal}\nFailure: {failure_description}\nWhy did this fail?"
        explanation = self.llm.generate(prompt)
        return self.extract_root_cause(explanation)

// Re-Planner Module - Generates corrected plans
class InteractiveRePlanner:
    """Generates corrected plans using feedback and explanations"""
    
    def replan(self, original_plan, failure_explanation, current_inventory):
        """Create improved plan based on failure analysis"""
        prompt = f"""
        Original plan failed. Reason: {failure_explanation}
        Current inventory: {current_inventory}
        Please fix the plan and provide corrected steps.
        """
        new_plan = self.llm.generate_code_plan(prompt)
        return new_plan

// Goal Selector with Horizon Prediction
class HorizonPredictiveSelector:
    """Learns to predict and rank completion horizons for parallel goals"""
    
    def __init__(self, state_dim, goal_dim):
        self.predictor = nn.Sequential(
            nn.Linear(state_dim + goal_dim, 256),
            nn.ReLU(),
            nn.Linear(256, 1)
        )
    
    def predict_horizon(self, state, goal):
        """Predict steps needed to complete goal from current state"""
        combined = torch.cat([state, goal], dim=-1)
        horizon = self.predictor(combined)
        return torch.exp(horizon) # Ensure positive horizon
    
    def select_best_goal(self, state, candidate_goals):
        """Select goal with minimal predicted horizon"""
        horizons = [self.predict_horizon(state, goal) for goal in candidate_goals]
        best_idx = torch.argmin(torch.tensor(horizons))
        return candidate_goals[best_idx]

// Complete DEPS Framework
class DEPS_Planner:
    """Describe, Explain, Plan and Select framework for open-world planning"""
    
    def __init__(self, llm_model, goal_selector, max_rounds=5):
        self.descriptor = ExecutionDescriptor()
        self.explainer = FailureExplainer(llm_model)
        self.replanner = InteractiveRePlanner(llm_model)
        self.selector = goal_selector
        self.max_rounds = max_rounds
    
    def plan_and_execute(self, task, initial_state):
        """Main DEPS planning and execution loop"""
        current_plan = self.generate_initial_plan(task)
        
        for round_num in range(self.max_rounds):
            # Execute plan with goal selection
            success, failed_step, current_state = self.execute_with_selection(
                current_plan, initial_state)
            
            if success:
                return current_plan, current_state
            
            # Interactive correction process

                current_plan, failed_step, current_state)

            current_plan = self.replanner.replan(current_plan, explanation, current_state)
            
        return current_plan, current_state # Return best plan found

        parallel_goals = self.identify_parallel_goals(plan)
        if parallel_goals:
            best_goal = self.selector.select_best_goal(state, parallel_goals)

SnapMoGen: Human Motion Generation from Expressive Texts

Chuan Guo1†, Inwoo Hwang2, Jian Wang1, Bing Zhou1†
1Snap Inc., 2Seoul National University
†Project Lead
One-Sentence Problem: Current text-to-motion datasets lack fine-grained, expressive descriptions and long-term continuity, limiting the generation of complex motions from casual user prompts.
一句话问题：当前的文本到动作数据集缺乏细粒度的、富有表现力的描述和长期连续性，限制了从随意的用户提示生成复杂动作的能力。

Contributions（贡献）

  • SnapMoGen Dataset: Introduced a large-scale dataset with 20K motion clips and 122K expressive text descriptions (avg. 48 words vs 12 in HumanML3D), preserving original temporal continuity.
    SnapMoGen 数据集：引入了一个大规模数据集，包含 20K 动作片段和 122K 富有表现力的文本描述（平均 48 个词 vs HumanML3D 的 12 个词），保留了原始的时间连续性。
  • MoMask++: Enhanced generative masked modeling by transforming motion into multi-scale token sequences and using a single unified transformer for all scales.
    MoMask++：通过将动作转换为多尺度 token 序列并使用单个统一的 Transformer 处理所有尺度，增强了生成式掩码建模。
  • Casual Prompt Handling: Demonstrated a pipeline using LLMs to reformat casual user prompts to align with the expressive narration style of SnapMoGen.
    随意提示处理：展示了一个使用 LLM 重新格式化随意用户提示的流程，以使其与 SnapMoGen 的表达性叙述风格保持一致。

Intro Figure（示例）

Challenges（挑战）

  • Dataset Constraints: Existing datasets contain short, clipped motions with simple descriptions, which hinders the model's ability to learn fine-grained control and long-term planning.
    数据集限制：现有的数据集包含简短的、剪辑过的动作和简单的描述，这阻碍了模型学习细粒度控制和长期规划的能力。
  • Model Capacity: Previous multi-stage generation (like MoMask) uses separate transformers, which might be inefficient for capturing multi-scale dependencies.
    模型容量：之前的多阶段生成（如 MoMask）使用单独的 Transformer，这在捕捉多尺度依赖关系方面可能效率低下。

Method（解决方法）

  1. Multi-Scale Tokenization: Represent motion as a sequence of tokens across multiple scales (coarse to fine) instead of separate residual layers.
    多尺度分词：将动作表示为跨越多个尺度（从粗到细）的 token 序列，而不是单独的残差层。
  2. Unified Generative Masked Transformer: A single transformer model learns to generate tokens at all scales simultaneously using a specific masking strategy.
    统一生成式掩码 Transformer：单个 Transformer 模型使用特定的掩码策略同时学习生成所有尺度的 token。
  3. LLM Refinement: Use Large Language Models to rewrite short user prompts into detailed, expressive narratives that match the training data distribution.
    LLM 优化：使用大型语言模型将简短的用户提示重写为详细、富有表现力的叙述，以匹配训练数据分布。

Experiments（实验）

  • Performance: MoMask++ achieves state-of-the-art results on both HumanML3D and the new SnapMoGen benchmark.
    性能：MoMask++ 在 HumanML3D 和新的 SnapMoGen 基准测试中均取得了最先进的结果。
  • Long-term Generation: The model effectively generates long, coherent motion sequences, benefiting from the dataset's preserved temporal continuity.
    长期生成： 得益于数据集保留的时间连续性，该模型能够有效地生成长而连贯的动作序列。

Discussion（讨论）

  • Data-Centric AI: The significant improvement highlights the importance of data quality (expressive text, continuous motion) over purely architectural changes.
    以数据为中心的 AI： 显著的改进凸显了数据质量（富有表现力的文本、连续动作）比纯粹的架构变更更为重要。

High-Level Insights: Why it Works?

SnapMoGen works by closing the gap between the complexity of human language and the complexity of human motion. By providing richer textual descriptions, the model learns a more robust mapping between semantics and kinematics. The unified MoMask++ architecture then efficiently models this relationship across different levels of motion detail, allowing for the generation of intricate and long-duration movements that were previously difficult to synthesize.
SnapMoGen 通过弥合人类语言复杂性与人体运动复杂性之间的差距而发挥作用。通过提供更丰富的文本描述，模型学习了语义和运动学之间更稳健的映射。统一的 MoMask++ 架构随后有效地在不同级别的运动细节上对这种关系进行建模，从而允许生成以前难以合成的复杂和长时间的运动。

Core Code Implementation（核心代码）

Conceptual implementation of MoMask++ showing multi-scale token handling in a single transformer.
MoMask++ 的概念实现，展示了单个 Transformer 中的多尺度 token 处理。 
class MoMaskPlusPlus(nn.Module):
    def forward(self, motion_ids, text_embed):
        # motion_ids: [Batch, Scales, SeqLen] (Multi-scale tokens)
        
        # 1. Flatten and Embed all scales
        x = self.token_emb(motion_ids.flatten(1, 2)) 
        
        # 2. Add Scale-Aware Positional Embeddings
        x = x + self.scale_pos_emb
        
        # 3. Unified Transformer Processing
        x = torch.cat([text_embed, x], dim=1)
        output = self.transformer(x)
        
        # 4. Predict tokens for all scales simultaneously
        logits = self.head(output)
        
        return logits

Methodï¼ˆæ–¹æ³•ï¼‰

The least-to-most prompting method enables complex reasoning through systematic problem decomposition:

  1. Problem Decomposition Stage: Query the language model to break down the complex problem into a series of simpler, sequential subproblems that can be solved step-by-step.
  2. Sequential Subproblem Solving: Solve each decomposed subproblem in sequence, where answers from previously solved subproblems are incorporated into subsequent prompts to facilitate the next reasoning step.
  3. Two-Stage Architecture: Maintain separate prompt contexts for decomposition (teaching how to break down problems) and solution (teaching how to solve subproblems sequentially).
  4. Answer Propagation: Use solutions from earlier subproblems as context for solving later subproblems, enabling cumulative reasoning that builds upon intermediate results.
  5. Few-shot Learning: Both decomposition and solving stages use few-shot prompting with exemplars demonstrating the desired behavior, requiring no model fine-tuning.

Discussionï¼ˆè®¨è®ºï¼‰

Why it Works?

Least-to-Most prompting succeeds by enabling systematic problem decomposition and sequential reasoning:

  • Problem Decomposition: Breaking complex problems into manageable subproblems allows language models to apply familiar reasoning patterns to novel, harder problems that exceed the complexity of training examples.
  • Answer Propagation: Using solutions from earlier subproblems as context for later reasoning creates a cumulative reasoning process that maintains coherence across multiple interdependent steps.
  • Sequential Dependencies: The method naturally handles problems with sequential dependencies where solving one subproblem unlocks the ability to solve subsequent ones, overcoming limitations of parallel reasoning approaches.
  • Compositional Generalization: By learning to decompose and recompose problems, the model can generalize to novel combinations of seen elements, enabling performance on tasks that require systematic composition.
  • Error Isolation: Decomposing problems into independent subproblems isolates errors to specific reasoning steps, preventing single mistakes from cascading through the entire reasoning chain.

Conclusionï¼ˆç»“è®ºï¼‰

This pioneering work introduces least-to-most prompting, a powerful prompting strategy that overcomes the limitations of chain-of-thought prompting by enabling systematic decomposition of complex problems into simpler subproblems solved sequentially. The method demonstrates remarkable generalization capabilities, achieving near-perfect performance on challenging compositional generalization benchmarks using minimal exemplars. By incorporating answers from previously solved subproblems into subsequent reasoning steps, least-to-most prompting creates a cumulative reasoning process that maintains coherence across multiple interdependent steps. Experimental results across symbolic manipulation, compositional generalization, and mathematical reasoning tasks show consistent superiority over chain-of-thought prompting, particularly on problems requiring more reasoning steps than demonstrated in the prompts. The approach represents a significant advancement in prompting techniques, offering a pathway toward more sophisticated reasoning capabilities in large language models without requiring architectural changes or extensive fine-tuning.

# Least-to-Most Prompting Implementation

// Two-stage prompting: Decomposition + Sequential Solving
def least_to_most_prompt(complex_question, decomposition_examples, solving_examples):
    """Implement least-to-most prompting strategy"""
    
    # Stage 1: Problem Decomposition
    decomposition_prompt = build_decomposition_prompt(decomposition_examples, complex_question)
    subproblems = query_language_model(decomposition_prompt)
    subproblem_list = parse_subproblems(subproblems)
    
    # Stage 2: Sequential Subproblem Solving
    solutions = {}
    for i, subproblem in enumerate(subproblem_list):
        # Build context with previously solved subproblems
        context = build_context_with_solutions(solutions)
        solving_prompt = build_solving_prompt(solving_examples, context, subproblem)
        solution = query_language_model(solving_prompt)
        solutions[subproblem] = solution
    
    # Final answer is the solution to the original complex question
    return solutions.get(complex_question, "Unable to solve")

// Example decomposition prompt
decomposition_examples = [
    {
        "question": "How many apples do Elsa and Anna have together?",
        "decomposition": "To answer 'How many apples do Elsa and Anna have together?', we need to know: 'How many apples does Anna have?', 'How many apples does Elsa have?'."
    }
]

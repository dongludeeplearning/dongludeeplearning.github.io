Paper Digest • ICCV 2025

Human-Object Interaction from Human-Level Instructions

Zhen Wu, Jiaman Li, Pei Xu, C. Karen Liu
Stanford University
One-Sentence Problem: How to synthesize physically plausible, long-horizon human-object interactions (full-body + fingers) from high-level language instructions in complex environments?
一句话问题：如何从复杂的环境中的高级语言指令合成物理上合理的、长时程的人-物交互（全身+手指）？

Contributions（贡献）

  • Unified HOI System: Proposed the first complete system for synthesizing physically plausible, long-horizon human-object interactions from high-level instructions.
    统一的 HOI 系统：提出了第一个完整的系统，用于从高级指令合成物理上合理的、长时程的人-物交互。
  • Synchronized Full-Body & Fingers: Achieved seamless coordination between full-body movements and detailed finger interactions, overcoming limitations of prior datasets.
    同步全身与手指：实现了全身运动与精细手指交互之间的无缝协调，克服了先前数据集的限制。
  • Physics-Based Tracking: Employed RL-based policy to track generated kinematic motions in physics simulation, ensuring physical plausibility and robustness.
    基于物理的跟踪：采用基于强化学习的策略在物理模拟中跟踪生成的运动学运动，确保物理合理性和鲁棒性。

Intro Figure（示例）

Challenges（挑战）

  • High-Level Instruction Grounding: Translating abstract instructions (e.g., "set up workspace") into precise object arrangements and execution plans requires commonsense reasoning.
    高级指令落地：将抽象指令（例如，“设置工作区”）转换为精确的对象排列和执行计划需要常识推理。
  • Data Scarcity: Lack of datasets containing synchronized full-body locomotion and detailed finger manipulation for large objects.
    数据稀缺：缺乏包含大型物体的同步全身运动和精细手指操作的数据集。

Method（解决方法）

  1. LLM Planning: Use LLMs to interpret instructions and predict object spatial relationships, then compute precise poses instead of direct prediction.
    LLM 规划：使用 LLM 解释指令并预测物体空间关系，然后计算精确姿态而不是直接预测。
  2. Hierarchical Motion Synthesis: 1. Generate full-body & object motion; 2. Inpaint finger motion based on object/wrist guidance; 3. Refine full-body motion conditioned on fingers.
    分层运动合成：1. 生成全身和物体运动；2. 基于物体/手腕引导修复手指运动；3. 以手指为条件细化全身运动。
  3. Physics Tracking via RL: Train a policy using PPO with a composite reward function (pose, hand-object distance, energy) to track kinematic motions in simulation.
    通过 RL 进行物理跟踪：使用具有复合奖励函数（姿态、手-物体距离、能量）的 PPO 训练策略，以在模拟中跟踪运动学运动。

Overview Figure（示意图）

High-Level LLM Planner (Sec. 4)

Overview: To address the challenge that LLMs struggle with precise 3D coordinate prediction, the paper proposes a two-stage pipeline (Sec 4.1 & 4.2) that uses "Spatial Relationships" as a robust intermediate representation.
概述：为了解决 LLM 难以直接预测精确 3D 坐标的问题，论文提出了一个两阶段流程（第 4.1 和 4.2 节），使用“空间关系”作为稳健的中间表示。
  • 4.1 LLM-Based Reasoning (Symbolic Stage):
    Instead of coordinates, the LLM is prompted to output a Scene Graph. It identifies relevant objects and predicts semantic spatial relationships (e.g., On(Laptop, Table), Right(Chair, Table)). This converts the "Instruction Understanding" problem into a "Constraint Generation" problem, leveraging the LLM's commonsense knowledge.
    4.1 基于 LLM 的推理（符号阶段）：
    LLM 不直接输出坐标，而是被提示输出一个场景图。它识别相关物体并预测语义空间关系（例如 On(Laptop, Table)，Right(Chair, Table)）。这利用 LLM 的常识知识，将“指令理解”问题转化为“约束生成”问题。
  • 4.2 Layout Optimization (Geometric Stage):
    The symbolic spatial constraints are converted into a differentiable loss function. An optimization algorithm then iteratively adjusts the 6DoF poses of objects to minimize this loss. This "Grounding" process ensures that the final layout is not only semantically correct but also geometrically valid (e.g., no inter-penetration, correct alignment).
    4.2 布局优化（几何阶段）：
    符号空间约束被转换为可微的损失函数。然后，优化算法迭代调整物体的 6DoF 姿态以最小化该损失。这个“落地”过程确保最终布局不仅在语义上正确，而且在几何上有效（例如，无穿透，正确对齐）。
  • Synthesis: This decoupled approach combines the semantic versatility of LLMs with the geometric precision of optimization, enabling the handling of complex, open-ended instructions.
    综合分析：这种解耦方法结合了 LLM 的语义通用性和优化的几何精度，使其能够处理复杂的、开放式的指令。

Experiments（实验）

  • Baselines: Compared against CHOIS (text-to-motion) + GRIP (grasp synthesis) pipeline.
    基线：与 CHOIS（文本到动作）+ GRIP（抓取合成）流程进行了比较。
  • Results: Outperformed baselines in synthesis quality and physical plausibility. Human studies confirmed the method generates more realistic interactions.
    结果：在合成质量和物理合理性方面优于基线。人体研究证实该方法生成了更逼真的交互。

Discussion（讨论）

  • Importance of Physics: Kinematic generation alone is insufficient for complex interactions; physics-based tracking is crucial to fix contacts and ensure realism.
    物理的重要性： 仅靠运动学生成不足以处理复杂的交互；基于物理的跟踪对于修复接触和确保真实感至关重要。
  • Hierarchical Generation: Breaking down the problem (Body -> Fingers -> Refinement) is an effective strategy to bypass the lack of paired high-quality data.
    分层生成： 分解问题（身体 -> 手指 -> 细化）是绕过缺乏配对高质量数据问题的有效策略。

High-Level Insights: Why it Works?

The system works by effectively bridging the semantic gap (LLM planning) and the motor control gap (Hierarchical Diffusion + RL Tracking). By decoupling finger generation from full-body motion, it leverages the best available data for each modality. The final physics tracking step acts as a "reality filter," grounding the generated motion in physical laws to eliminate artifacts like penetration or skating.
该系统通过有效地弥合语义差距（LLM 规划）和运动控制差距（分层扩散 + RL 跟踪）来发挥作用。通过将手指生成与全身运动解耦，它利用了每种模态的最佳可用数据。最后的物理跟踪步骤充当“现实过滤器”，将生成的运动建立在物理定律之上，以消除穿透或滑动等伪影。

Core Code Implementation（核心代码）

Reward function logic for RL-based physics tracking, penalizing deviation from kinematic reference.
基于 RL 的物理跟踪的奖励函数逻辑，惩罚与运动学参考的偏差。 
def compute_reward(self, body_diff, hand_obj_diff, energy):
    # 1. Full-body Tracking Reward
    r_body = 0.5 * torch.exp(-15 * body_diff.rot) + \
             0.5 * torch.exp(-15 * body_diff.pos)
             
    # 2. Hand-Object Interaction Reward
    # Encourages precise finger placement relative to object
    r_hand = torch.exp(-5 * hand_obj_diff)
    
    # 3. Energy Penalty (Smoothness)
    r_energy = torch.exp(-1/900 * energy)
    
    # Weighted Sum
    return 0.8 * r_body + 0.2 * r_hand + 0.05 * r_energy

Methodï¼ˆæ–¹æ³•ï¼‰

The Tree of Thoughts framework enables deliberate problem solving by structuring reasoning as tree search over thoughts:

  1. Thought Decomposition: Break down the problem-solving process into coherent units of text (thoughts) that represent meaningful intermediate steps, such as equations in math problems or paragraphs in writing tasks.
  2. Thought Generation: Use LM prompts to generate multiple candidate thoughts from each state, either by sampling independently or proposing thoughts sequentially while maintaining diversity.
  3. State Evaluation: Evaluate the quality and progress of different thought states using LM-based heuristics, either by valuing states independently or voting across states to determine the most promising paths.
  4. Search Algorithms: Apply BFS for shallow, breadth-focused exploration or DFS for deep, depth-focused exploration with pruning and backtracking capabilities.
  5. Integration: Combine all components into a cohesive framework where thoughts serve as nodes in a tree, and the LM provides both generation and evaluation capabilities for deliberate decision-making.

Why it Works?

Tree of Thoughts succeeds by bridging the gap between fast System 1 and deliberate System 2 thinking in language models:

  • Multi-Path Exploration: Unlike linear Chain-of-Thought, ToT explores multiple reasoning trajectories simultaneously, allowing the model to discover diverse approaches and avoid getting stuck in suboptimal paths.
  • Self-Evaluation and Deliberation: The framework incorporates LM-based evaluation mechanisms that enable the model to assess its own progress and make informed decisions about which paths to pursue or abandon.
  • Search Algorithm Integration: By combining classical search algorithms (BFS/DFS) with LM capabilities, ToT provides systematic exploration with backtracking, enabling the model to handle complex problems that require planning and strategic thinking.
  • Thought-Level Reasoning: Operating at the level of coherent thoughts rather than individual tokens allows for more meaningful decision-making and evaluation, capturing higher-level reasoning patterns.
  • Adaptive Problem Solving: The modular design allows ToT to adapt to different problem types by varying thought decomposition, generation strategies, and search algorithms, making it applicable across diverse domains.

Conclusionï¼ˆç»“è®ºï¼‰

This groundbreaking work introduces Tree of Thoughts, a revolutionary framework that transforms language models from mere text generators into deliberate problem solvers capable of strategic planning and search. By conceptualizing problem solving as tree search over coherent units of text (thoughts), the authors bridge the gap between classical artificial intelligence approaches and modern language models, enabling LMs to perform the kind of systematic exploration and decision-making previously only possible with specialized algorithms. The framework's modular designâ€”allowing independent variation of thought decomposition, generation, evaluation, and search algorithmsâ€”provides unprecedented flexibility and adaptability across diverse problem domains. Experimental results on three challenging tasks demonstrate the framework's power: achieving a 74% success rate on Game of 24 where Chain-of-Thought failed at 4%, solving Mini Crosswords with 78% letter-level accuracy, and generating more coherent creative writing. ToT represents a fundamental advancement in how we interact with and deploy language models, opening new possibilities for complex reasoning, planning, and decision-making applications. The work establishes a new paradigm for language model inference that combines the creativity and knowledge of LMs with the systematicity and deliberation of classical search algorithms, potentially enabling more sophisticated AI systems that can tackle problems requiring genuine intelligence and strategic thinking.

// BFS search algorithm with thought generation and evaluation
def tot_bfs(problem, lm, thought_generator, state_evaluator, max_steps=3, beam_width=5):
    """Tree of Thoughts with Breadth-First Search"""
    
    root_state = (problem, []) # (current_problem, thought_history)
    queue = [root_state]
    
    for step in range(max_steps):
        candidates = []
        
        # Generate thoughts for each state in current level
        for state in queue:
            new_thoughts = thought_generator.generate(state, num_candidates=beam_width)
            candidates.extend([(state, thought) for thought in new_thoughts])
        
        # Evaluate all candidate states
        evaluated_states = []
        for state, thought in candidates:
            new_state = (update_problem(state[0], thought), state[1] + [thought])
            score = state_evaluator.evaluate(new_state)
            evaluated_states.append((new_state, score))
        
        # Keep top beam_width states for next level
        evaluated_states.sort(key=lambda x: x[1], reverse=True)
        queue = [state for state, _ in evaluated_states[:beam_width]]
    
    # Return the best final state
    best_state = max(queue, key=lambda s: state_evaluator.evaluate(s))
    return extract_solution(best_state)

// DFS search with pruning and backtracking
def tot_dfs(state, step, max_steps, thought_generator, state_evaluator, threshold=0.3):
    """Tree of Thoughts with Depth-First Search"""
    
    if step >= max_steps:
        return extract_solution(state)
    
    candidates = thought_generator.generate(state, num_candidates=5)
    
    for thought in candidates:
        new_state = (update_problem(state[0], thought), state[1] + [thought])
        score = state_evaluator.evaluate(new_state)
        
        # Prune if state evaluation below threshold
        if score < threshold:
            continue
        
        # Recurse deeper
        result = tot_dfs(new_state, step + 1, max_steps, thought_generator, state_evaluator, threshold)
        if result is not None:
            return result
    
    return None # No solution found
